This is the API of the Plealog Bioinformatics BLAST Filter software.

What is a filter?

A filter is made of some rules, each of them being on constraint applied on the data contains in a BLAST result file. This library is capable of reading legacy NCBI XML file (-outfmt 5 argument from BLAST+; -m 7 for legacy BLAST) and creating an object-based data model in memory. This model lets you access the many data fields (Hit ID, alignment length, e-value, scores, etc.) of a BLAST result. So, you use a filter to select relevant hits using your own constraints.

How to use it? (... for the developers)

This is a library, so its primary purpose is to provide you with a way of programming your own tools relying on the library.

Simply have a look at the test code, TestFilterSystem.java, located in the test package. There, you'll see how to read an NCBI XML result file, filter it and save results in a new file.

How to use it? (... for the end-users)

Now, for the end users, I have setup the BLAST Filter Tool providing a very easy-to-use command line software. This data filtering engine is also used in the BlastViewer software.
